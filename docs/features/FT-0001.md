## Summary

Every hop-top CLI built with kit/cli behaves identically — same flags,
same help format, same error handling, same version output. Language is
an implementation detail, not a UX difference.

## Acceptance criteria

- [ ] `--help` flag works at every level; no `help` subcommand
- [ ] `--version` prints `<tool> <version>` to stdout
- [ ] `--format table|json|yaml` supported globally
- [ ] `--quiet` suppresses non-essential output
- [ ] `--no-color` disables ANSI
- [ ] Errors to stderr with non-zero exit
