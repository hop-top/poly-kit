# Run the kit engine

> Audience: end-users adopting kit who need to start the engine
> sidecar and connect a tool to it. For background on what the
> engine is and when you need it, read
> [engine-overview.md](../concepts/engine-overview.md) first.

This page gets you from "engine not running" to "engine running and
serving requests" in five steps.

## Prerequisites

- `kit` binary installed and on `$PATH`. If not yet:
  `go install hop.top/kit/cmd/kit@latest` or
  `brew install hop-top/tap/kit`. Verify with `kit --version`.
- A directory you want the engine to use for state. Anywhere
  writable; example below uses `./engine-data`.

## 1. Start the engine

```bash
kit serve --port 8080 --data ./engine-data
```

What happens:

- A SQLite database is created at `./engine-data/documents.db`.
- An Ed25519 identity keypair is generated at
  `./engine-data/identity/{public,private}.pem` if one doesn't
  already exist.
- The engine binds `127.0.0.1:8080` and logs ready to stderr.

`--port 0` (the default) picks a random free port; useful when
multiple engines run on one host. `--data` defaults to `./data`
relative to the working directory.

## 2. Verify it's up

```bash
curl http://localhost:8080/health
```

Expected: HTTP 200 with a JSON body containing the engine version
and identity fingerprint. Non-200 means the engine isn't ready —
check the `kit serve` log line.

## 3. Point a tool at it

The engine exposes localhost HTTP. How a tool connects depends on
its language:

- **Go (kit-linked):** the tool typically uses kit's libraries
  in-process and only talks to the engine for cross-process
  features (sync, peer mesh). Set `KIT_ENGINE_URL=http://localhost:8080`
  for code paths that look it up.
- **TypeScript / Python (SDK):** the SDKs spawn the engine for
  you on first use. If an engine is already running, set
  `KIT_ENGINE_URL` to skip the spawn.
- **Other languages:** call the HTTP API directly. The endpoints
  live under `/documents`, `/sync`, `/identity`, `/peers`,
  `/events` (WebSocket). See
  [engine-protocol.md](../reference/engine-protocol.md) for the wire format.

## 4. Confirm a tool can reach it

End-to-end smoke check (replace `<your-tool>` with your CLI's
binary):

```bash
KIT_ENGINE_URL=http://localhost:8080 <your-tool> --help
```

If the tool relies on engine state (documents, identity), exercise
the relevant subcommand and confirm it returns rather than erroring
on connection refused.

## 5. Stop the engine

```bash
kill $(cat ./engine-data/kit-engine.pid)
# or simply Ctrl-C the foreground kit serve process
```

The PID file is written when `--daemon` is used. State on disk
(`./engine-data/`) survives — the next `kit serve` on the same
data dir resumes with the same identity and documents.

## Daemon mode

For multiple kit-using tools on one host that should share an
engine, run it as a daemon:

```bash
kit serve --daemon --port 8080 --data ~/.kit
```

Tools find the running engine via the PID file at
`~/.kit/kit-engine.pid` (each PID file records its port). Set
`KIT_ENGINE_URL` to override the discovery path.

## Encryption at rest

Add `--encrypt` to encrypt the SQLite document store with a key
derived from the engine's identity:

```bash
kit serve --port 8080 --data ./engine-data --encrypt
```

Don't toggle this flag on an existing data dir without re-keying
first — see [engine-security.md](../reference/engine-security.md) for the
threat model. A how-to for the rekey flow lives at
[encrypt-engine-data.md](encrypt-engine-data.md) (planned, see
the engine-security.md "Related" section).

## Troubleshooting

- **"address already in use"** — another engine (or another
  process) holds the port. Pick a different `--port`, or
  `lsof -i :8080` to find the offender.
- **Tool says "engine unreachable"** — confirm `curl
  localhost:8080/health` works from the same host as the tool.
  Check `KIT_ENGINE_URL` is set or the SDK's spawn path is
  enabled.
- **Identity mismatch on restart** — the engine refuses to start
  if `--data` points at a directory whose identity keypair was
  generated by a different engine version. Rotate keys with the
  process described in [engine-security.md](../reference/engine-security.md).
